# फाइल व्यवस्थापन प्रणाली (गाउँपालिका फाइल ट्र्याकिङ सिस्टम)

शाखा-शाखा बीच फाइल/कागजात कुन कर्मचारी, जनप्रतिनिधि, संघसंस्था वा सेवाग्राहीले
कहिले (वि.सं./ई.सं. मिति) बुझेको र फिर्ता गरेको हो भन्ने कुरा **QR कोड स्क्यान** मार्फत
ट्र्याक गर्ने वेब-आधारित प्रणाली।

## मुख्य सुविधाहरू
- फाइल बुझाउँदा स्वतः **QR कोड** जनरेट (प्रिन्ट गरी फाइलसँगै टाँस्न मिल्ने)
- क्यामेराबाट **QR स्क्यान** गरेर "फाइल लगेको" / "फाइल फिर्ता भएको" रेकर्ड
- कार्यालयको नाम, लोगो, **शाखा** र **वर्ग (Category)** सेटिङबाट व्यवस्थापन
- बुझ्ने पक्ष: कर्मचारी / जनप्रतिनिधि / संघसंस्था / सेवाग्राही — किसिम छुट्याएर राख्ने
- **वि.सं. (Nepali BS) र ई.सं.** दुवै मिति देखाउने
- फिल्टर सहितको पूर्ण अभिलेख (History) + CSV एक्सपोर्ट
- JSON ब्याकअप/रिस्टोर
- **Firebase Firestore** मा जडान गर्न मिल्ने (धेरै प्रयोगकर्ता, बहु-यन्त्र sync-का लागि)
- GitHub Pages मार्फत नि:शुल्क डिप्लोय गर्न मिल्ने (स्थिर HTML/JS भएकाले)

## स्थानीय रूपमा चलाउने
कुनै build step चाहिँदैन — शुद्ध HTML/CSS/JS हो।
```bash
# कुनै पनि static server, जस्तै:
npx serve .
# वा
python3 -m http.server 8080
```
त्यसपछि ब्राउजरमा `http://localhost:8080` खोल्नुहोस्।

> ⚠️ QR क्यामेरा स्क्यानका लागि ब्राउजरले HTTPS वा `localhost` मात्र क्यामेरा अनुमति दिन्छ।

## Firebase जडान गर्ने (धेरै कम्प्युटर/शाखाबीच एउटै डेटा हेर्न)
1. [Firebase Console](https://console.firebase.google.com) मा नयाँ Project बनाउनुहोस्।
2. **Build → Firestore Database → Create database** (Production mode) सुरु गर्नुहोस्।
3. **Project settings → General → Your apps → Web (</>) → Register app**।
4. देखिने `firebaseConfig` कपी गरी `firebase-config.js` मा पेस्ट गर्नुहोस्।
5. `firebase-config.js` मा `USE_FIREBASE = true` गर्नुहोस्।
6. Firestore Rules (Rules ट्याबमा) — सुरुमा testing-का लागि:
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /{document=**} {
         allow read, write: if request.auth != null; // लगइन गरेकालाई मात्र
       }
     }
   }
   ```
   वास्तविक प्रयोगका लागि Firebase Authentication (Email/Password) पनि सक्षम गरी
   कर्मचारीहरूलाई लगइन दिनुहोस् — हाल यो टेम्प्लेटमा Auth UI जोडिएको छैन,
   चाहिएमा थप्न सकिन्छ (Firebase Auth SDK यो प्रोजेक्टमा already included छ)।

## GitHub मा राखेर नि:शुल्क Deploy गर्ने (GitHub Pages)
```bash
cd file-tracking-system
git init
git add .
git commit -m "Initial commit: गाउँपालिका फाइल व्यवस्थापन प्रणाली"
git branch -M main
git remote add origin https://github.com/<तपाईंको-username>/<repo-नाम>.git

```
त्यसपछि:
1. GitHub repo → **Settings → Pages**
2. **Source**: "Deploy from a branch" → Branch: `main`, folder: `/ (root)`
3. केही सेकेन्डमा `https://<username>.github.io/<repo-नाम>/` मा साइट लाइभ हुन्छ।

## ⚡ फास्ट-ट्र्याक (quick.html) — १ मिनेटमा, नपढेकाले पनि प्रयोग गर्न मिल्ने
यो सबैभन्दा सजिलो मोड हो — ठूला आइकन/फोटो-टाइल छानेर मात्र काम हुन्छ, धेरै टाइप गर्नु पर्दैन,
र नाम बोलेर पनि भर्न मिल्छ (आवाजबाट)। पढ्न-लेख्न कठिन हुनेले पनि प्रयोग गर्न सकून् भनेर
यसरी डिजाइन गरिएको छ:

- **लगइन**: नाम पढ्नु पर्दैन — आफ्नो फोटो/initial-avatar भएको टाइलमा एकपटक थिच्ने, त्यसपछि
  PIN भए ठूलो नम्बर-प्याडबाट छाप्ने (कुनै किबोर्ड टाइप गर्नु पर्दैन)।
- **कारबाही**: दुई मात्र ठूला बटन — "📤 फाइल बुझाउने" र "📥 फाइल फिर्ता"।
- **शाखा/वर्ग छान्ने**: आइकन-टाइलमा थिचेर मात्र (पढ्न नजाने पनि आइकनले चिन्न सकिन्छ)।
- **फ्रन्ट पेज फोटो**: एक थिचाइमा क्यामेरा खुल्छ।
- **बुझ्ने व्यक्तिको नाम**: (क) पहिले नै आएका व्यक्तिको फोटो/नाम-टाइलमा थिच्ने, वा
  (ख) 🎤 "बोलेर नाम भन्नुहोस्" थिचेर आवाजले नाम भन्ने (Speech-to-Text — Chrome/Android मा
  राम्रो काम गर्छ), वा (ग) आवश्यक परे मात्र टाइप गर्ने विकल्प पनि छ।
- **पेश गर्ने**: एउटै ठूलो हरियो बटन — त्यसपछि फिर्ता स्वतः २४ घण्टा म्याद राखिन्छ,
  QR स्वतः बन्छ, र लगइन गरेकै कर्मचारीले लगत्तै अर्को फाइल पनि (फेरि लगइन नगरी) छिटो
  थप्न सक्छन्।
- **फिर्ता**: फ्रन्ट पेज फोटो मात्र खिच्ने → ठूला फोटो-कार्डमध्ये सहि फाइलमा थिच्ने → सकियो।

**कर्मचारीको फोटो थप्दा झन् सजिलो हुन्छ**: Settings → "कर्मचारी" मा नाम/शाखा/PIN का साथै
फोटो पनि (वैकल्पिक) राख्नुभयो भने फास्ट-ट्र्याकको लगइन स्क्रिनमा नाम पढ्नु नपरी सिधै
आफ्नो अनुहार चिनेर थिच्न मिल्छ — नपढेकाहरूका लागि सबैभन्दा ठूलो सहयोग यसैले गर्छ।

> फास्ट-ट्र्याकमा फाइलको शीर्षक/दर्ता नं. जस्ता विस्तृत विवरण छुटाइएको छ (गति बढाउन);
> विस्तृत विवरण चाहिने फाइलका लागि पूर्ण फारम (index.html) प्रयोग गर्नुहोस्।

## 👤 कर्मचारी लगइन र स्वतः-भरिने फारम — नयाँ सुविधा
अब फाइल बुझाउने प्रत्येक पटक कर्मचारीको नाम/शाखा म्यानुअल नटाइप गरी, एक पटक "लगइन"
गरेपछि फारम आफैं भरिन्छ।

**सेटअप (पहिलो पटक मात्र)**
1. Settings → "कर्मचारी (लगइन/Handover का लागि)" मा गई हरेक कर्मचारीको नाम, शाखा,
   र ४ अंकको PIN राखी थप्नुहोस्।
2. त्यसपछि प्रणाली खोल्दा (index.html वा scan.html) लगइन विन्डो देखिन्छ — नाम छानी
   PIN हालेपछि लगइन हुन्छ। यो सत्र सोही ब्राउजर/फोनमा सुरक्षित रहन्छ (फेरि लगइन गर्नु पर्दैन,
   "लगआउट" नथिचेसम्म)।

**लगइन पछि स्वतः हुने कुरा**
- फाइल बुझाउने फारममा **"पठाउने शाखा (बाट)"** र **"फाइल बुझाउने कर्मचारी (Handover गर्ने)"**
  स्वतः भरिन्छ र लक हुन्छ (आवश्यक परे छेउको "बदल्नुहोस्" थिचेर म्यानुअल बदल्न पनि मिल्छ)।
- मोबाइल स्क्यान (`scan.html`) मा पनि लगइन गरेपछि, कुन कर्मचारीले "लगेको/फिर्ता" स्क्यान
  गरेको हो भन्ने कुरा प्रत्येक फाइल रेकर्डमा (`takenScannedBy` / `returnScannedBy`) अभिलेख हुन्छ।

**बुझ्ने व्यक्ति (Carrier) पनि सजिलै — Self-learning डाइरेक्टरी**
- "बुझ्ने व्यक्तिको नाम" फिल्डमा पहिले नै आएको नाम टाइप गर्न थाल्नेबित्तिकै सुझाव देखिन्छ
  (autocomplete), र नाम छानेपिच्छे किसिम/सम्पर्क/शाखा स्वतः भरिन्छ।
- नयाँ (पहिलोपटक आउने) व्यक्तिको नाम भरे, फाइल दर्ता हुनासाथ त्यो नाम आफैं
  Settings → "लग्ने व्यक्ति डाइरेक्टरी" मा थपिन्छ — अर्को पटकदेखि उसैको लागि पनि autofill हुन्छ।
  म्यानुअल एन्ट्री गर्नै पर्दैन।

**फिर्ता बुझाउने अन्तिम मिति स्वतः २४ घण्टा**
- फाइल बुझाउँदा "फिर्ता बुझाउने अन्तिम मिति" फिल्ड स्वतः हालको समयबाट २४ घण्टापछिको
  मिति/समयमा भरिन्छ (आवश्यक परे म्यानुअल फेरबदल गर्न मिल्छ)। यसैका आधारमा ड्यासबोर्डमा
  "म्याद नाघेका" फाइल गणना हुन्छ।

> ⚠️ यो PIN-आधारित लगइन हलुका सुविधाजनक (convenience) सिस्टम हो, बैंकिङ-स्तरको सुरक्षित
> authentication होइन। कडा सुरक्षा चाहिने कार्यालयका लागि Firebase Authentication
> (Email/Password वा OTP) जोडेर यसलाई अझ बलियो बनाउन सकिन्छ — चाहनुभए थप्न सकिन्छ।

## 📷 फोटोबाट फिर्ता चिन्ने (QR प्रिन्ट नगरी) — नयाँ सुविधा
QR प्रिन्ट/टाँस्नुको सट्टा अझ सजिलो तरिका: फाइल बुझाउँदा फाइलको **फ्रन्ट पेजको फोटो**
मात्र खिचे पुग्छ; फिर्ता आउँदा सोही फ्रन्ट पेज फेरि खिचेपिच्छे प्रणालीले आफैं मिल्दो फाइल पत्ता लगाउँछ।

**कसरी काम गर्छ**
1. फाइल बुझाउने फारम (Desktop/index.html) मा "फ्रन्ट पेज फोटो" फिल्डमा क्यामेराले
   कागजातको पहिलो पेज खिच्नुहोस्। यसको एउटा सानो "visual fingerprint" (dHash) र
   thumbnail फाइल रेकर्डसँगै सुरक्षित हुन्छ (क्लाउडमा फोटो अपलोड हुँदैन, ब्राउजर/Firestore
   मै सानो हल्का hash + thumbnail मात्र बस्छ)।
2. फिर्ता गर्दा मोबाइल स्क्यान पृष्ठ (`scan.html`) मा **"📷 फिर्ता (फोटो)"** मोड छान्नुहोस्,
   फेरि सोही फ्रन्ट पेज खिच्नुहोस्।
3. प्रणालीले हाल "लगेको" स्थितिमा रहेका सबै फाइलको फोटोसँग तुलना गरी सबैभन्दा मिल्दो
   १-३ वटा (मिलान %-सहित) देखाउँछ — कर्मचारीले सही फाइल छानेर "फिर्ता गर्ने" थिच्नुपर्छ।

**सीमाहरू (जानिराख्नुहोस्)**
- यो image-hash आधारित हलुका मिलान हो, न कि precise OCR/AI पहिचान — उज्यालो/कोण
  धेरै फरक भए मिलान % कम आउन सक्छ, त्यसैले अन्तिम पुष्टि सधैं मान्छेले नै गर्ने राखिएको छ
  (स्वतः/बिना-पुष्टि फिर्ता कहिल्यै हुँदैन)।
- QR स्टिकर टाँस्नै नपरोस् भन्नेहरूका लागि यो राम्रो विकल्प हो; तर धेरै उस्तै देखिने
  फर्म/ढाँचा भएका कागजातहरूमा (सबै फाइलको फ्रन्ट पेज लगभग एउटै जस्तो देखिने भए)
  मिलान अझ राम्रो बनाउन फ्रन्ट पेजमा फाइल दर्ता नं./छाप जस्तो छुट्टिने चिन्ह देखिने गरी फोटो खिच्नुहोस्।
- चाहनुहुन्छ भने QR र फोटो-मिलान दुवै एकसाथ पनि प्रयोग गर्न मिल्छ (एउटा backup को रूपमा अर्को)।

## मोबाइलबाट QR स्क्यान गर्ने (scan.html)
शाखामा खटिएको कर्मचारीले ल्यापटपको साटो **मोबाइल फोनबाटै** सिधै QR स्क्यान गर्न मिल्ने
छुट्टै हल्का पृष्ठ `scan.html` राखिएको छ (ठूला बटन, पूरा-स्क्रिन क्यामेरा, टर्च/फ्ल्यास टगल)।

- मुख्य प्रणाली (`index.html`) को "QR स्क्यान" ट्याबमा पुगेर देखिने QR मोबाइलले
  स्क्यान गरे सिधै `scan.html` खुल्छ, वा लिङ्क सिधै सेयर पनि गर्न सकिनुहुन्छ:
  `https://<तपाईंको-डोमेन>/scan.html`
- फोनको ब्राउजरमा "Add to Home Screen" गरे एपजस्तै आइकनबाट खोल्न मिल्छ।
- क्यामेरा प्रयोगका लागि ब्राउजरले **HTTPS** (वा `localhost`) मात्र अनुमति दिन्छ —
  GitHub Pages वा Firebase Hosting मा deploy गरेपछि यो automatic हुन्छ।
- ⚠️ धेरै फोन/कम्प्युटरबाट एउटै अभिलेख (real-time sync) हेर्न/अपडेट गर्न चाहेमा
  **Firebase Firestore जडान (`USE_FIREBASE = true`) अनिवार्य छ** — नत्र हरेक
  यन्त्रको डेटा त्यही यन्त्रको ब्राउजरमा मात्र (localStorage) सीमित रहन्छ।

## वि.सं. मिति बारे नोट
`bs-date.js` ले हलुका/approximate AD→BS रूपान्तरण गर्छ (UI मा मिति देखाउनका लागि ठीक)।
आधिकारिक अभिलेखमा **ठ्याक्कै सही** वि.सं. पात्रो चाहिएमा `nepali-date-converter` जस्तो
maintained npm प्याकेज जडान गरी सोही फाइलका functions replace गर्नुहोस्:
```bash
npm install nepali-date-converter
```
